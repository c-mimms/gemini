# Project 2: Alternate Tech Tree Simulator - Expanded Design

## 1. System Architecture
The system consists of three main components:
*   **The State Manager:** Reads and writes `tech_tree.json`. It tracks the current year, active technologies, and deprecated technologies.
*   **The Inventor Agent:** Receives the current state and proposes N valid "next steps" (e.g., combining two active technologies, or discovering a new fundamental principle).
*   **The Historian Agent:** Takes the selected "next step" and generates the actual content (a press release, a Wikipedia article, or a magazine review of the new tech).

## 2. Tech Tree JSON Structure
The core state is a directed acyclic graph (DAG) stored in JSON:
```json
{
  "current_year": 1985,
  "nodes": {
    "tech_vacuum_tube": {
      "status": "obsolete",
      "discovered_year": 1940
    },
    "tech_analog_microchip": {
      "status": "active",
      "discovered_year": 1982,
      "parents": ["tech_vacuum_tube", "tech_miniaturization"]
    }
  }
}
```

## 3. Pruning Context Effectively
When feeding the prompt to the Inventor Agent, we cannot pass the entire JSON if it grows to hundreds of nodes.
*   **Obsolete nodes** are hidden from the context.
*   **Ubiquitous nodes** (technologies that are so common they are just assumed, e.g., "Electricity") are moved to a `background_assumptions.md` file, which is a static block of text.
*   The Inventor Agent only sees `status: active` nodes and `status: developing` nodes.

## 4. Web Display
The output website should feature a D3.js or React Flow interactive node graph. Clicking a node opens the generated article (written by the Historian Agent) detailing how that technology came to be.
