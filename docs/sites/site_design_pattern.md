# Creating a New AI-Generated Static Site

This document describes the end-to-end pattern used by all content-generation sites in this workspace. Follow this guide to create a new site from scratch.

---

## Architecture Overview

Each site consists of four components:

```
sites/
└── my_site/
    ├── src/                    # AI-generated article HTML files (the "raw content")
    ├── output/                 # Built site (generated by build script, gitignored)
    ├── build_my_site.py        # Build script: HTML → full site → S3 + CloudFront
    ├── my_site.css             # Site-wide stylesheet
    ├── my_site_task.md         # AI task prompt (the agent's "job description")
    └── data/                   # Optional: reference datasets, images, research
```

The agent produces raw HTML fragments → the build script assembles a full site → published to S3/CloudFront.

## Step 1: Site Directory

Create the directory structure:
```bash
mkdir -p /Users/chris/code/gemini/sites/my_site/{src,data}
```

## Step 2: The Task Definition File (`my_site_task.md`)

This file is the agent's complete prompt. It is referenced in the task scheduler as a `--context` file.

### Structure

```markdown
# Task: [Site Name] Content Generation

## Goal / Persona
One paragraph. Who is the agent? What is their expertise?
What is the site's purpose/audience?

## Data Directory (optional)
Point to any datasets: `/Users/chris/code/gemini/sites/my_site/data/`

## Step 1: Check Existing Content
The agent MUST check `src/` to avoid duplicate topics.
Tell them: list existing files, avoid repeating topics.

## Step 2: Pick Topic / Format
List any supported content formats (e.g. Deep Dive, Timeline, Data Analysis).
If multiple formats exist, tell the agent to semi-randomly select one.

## Step 3: Research
Describe what sources to use. Web search, dataset files, browser navigation, etc.

## Step 4: Write the Article

### Output Requirements
- Save to: `/Users/chris/code/gemini/sites/my_site/src/YYYY-MM-DD_short-slug.html`
- Format: raw HTML fragment — NO `<html>/<head>/<body>` tags
- Must start with a metadata block:

    ```html
    <div class="metadata" style="display:none;">
        <meta name="title" content="Article Title">
        <meta name="description" content="1-2 sentence summary.">
        <meta name="tag" content="Format | Theme">
    </div>
    ```

### CSS Tool Inventory
List every CSS class the agent is allowed to use.
Be prescriptive — agents that know exactly which classes to use produce better markup.

### Image Handling (if applicable)
```bash
curl -s -L -A "MyBot/1.0" "https://commons.wikimedia.org/wiki/Special:FilePath/FILENAME?width=1000" -o OUTPUT_PATH
file OUTPUT_PATH  # Delete if HTML/text — it means the download failed
```

## Step 5: Publish the Site
```bash
python3 /Users/chris/code/gemini/sites/my_site/build_my_site.py \
  --source /Users/chris/code/gemini/sites/my_site/src/ \
  --s3-bucket s3://gemini-designs-portfolio-2026-v2/my-site/ \
  --cloudfront-id [DISTRIBUTION_ID] \
  --site-name "My Site" \
  --site-tagline "A short tagline"
```

## Step 6: Log Completion
Tell the agent what to print: topic, filename saved, publish status.
```

## Step 3: The Build Script (`build_my_site.py`)

Copy from an existing script and adapt. The canonical interface is:

```python
parser = argparse.ArgumentParser()
parser.add_argument("--source",         required=True)   # src/ directory
parser.add_argument("--s3-bucket",      required=False)  # s3://bucket/prefix/
parser.add_argument("--cloudfront-id",  required=False)  # distribution ID
parser.add_argument("--site-name",      default="My Site")
parser.add_argument("--site-tagline",   default="A tagline")
```

### What the build script must do

1. **Parse article metadata** from each `src/*.html` file (read `<meta name="title">`, `<meta name="description">`, `<meta name="tag">`, and extract date from filename)
2. **Wrap each article** in a full HTML document with `<head>`, site header, site footer, and links to `my_site.css`
3. **Build `index.html`** — a homepage listing all articles (sorted newest-first)
4. **Copy CSS** from the site root into `output/`
5. **Sync to S3**: `aws s3 sync output/ s3://bucket/prefix/ --delete`
6. **Invalidate CloudFront**: `aws cloudfront create-invalidation --distribution-id ID --paths "/*"`

### Metadata extraction pattern

```python
# Pull from <meta name="title" content="...">
m = re.search(r'<meta[^>]+name=["\']title["\'][^>]+content=["\']([^"\']+)["\']', html, re.IGNORECASE)
# Pull date from filename: YYYY-MM-DD_slug.html
m = re.match(r'^(\d{4}-\d{2}-\d{2})_(.+)\.(html?)$', filename)
```

### Good examples to copy from
- **`build_news.py`** — most feature-complete; has CloudFront, search index, related articles sidebar
- **`build_museum.py`** — similar to news, with format-filter dropdown on index
- **`build_georgia_mining.py`** — data-focused variant with data directory sync
- **`build_prayer.py`** — simplest; minimal index, category filtering

## Step 4: The Stylesheet (`my_site.css`)

Every site has its own CSS file. The stylesheet is referenced by the build script and linked in every article's `<head>`.

### Conventions
- Define CSS custom properties (`--vars`) at `:root` for colors, fonts, spacing
- Use semantic class names that match the task definition's "CSS Tool Inventory" exactly
- Include: body, typography, site header/footer, article wrapper, article card (for the index), data tables, blockquote, references section
- Include responsive breakpoints at minimum `@media (max-width: 768px)`

---

## Step 5: Register the Site in the Task Scheduler

```bash
task add \
  --name "My Site Content" \
  --schedule "0 15 * * *" \
  --executor gemini \
  --context /Users/chris/code/gemini/sites/my_site/my_site_task.md
```

Verify with `task list` and `task show ID`.

---

## Step 6: Set Up CloudFront

1. Create a CloudFront distribution in AWS pointing to the S3 bucket with the site's path prefix as the origin path
2. Note the Distribution ID (format: `EXXXXXXXXXX`)
3. Add `--cloudfront-id [ID]` to the publish command in `my_site_task.md`

---

## Step 7: Checklist for a New Site

- [ ] Directory created: `sites/my_site/{src,data}`
- [ ] Task definition written: `my_site_task.md` with all 6 sections
- [ ] Build script created: `build_my_site.py` — adapted from an existing one
- [ ] Stylesheet created: `my_site.css`
- [ ] Task registered in scheduler with correct `--context` path
- [ ] CloudFront distribution created and ID added to task definition
- [ ] First test run: `task run ID` and verify site appears at CloudFront URL
- [ ] `output/` added to `.gitignore`
